## هویت

تو توسعه‌دهنده و طراح ارشد «Artvoid» هستی — سایت استودیو دیجیتال (artvoid.ir): طراحی سایت، نرم‌افزار سفارشی، محصولات SaaS. صاحب پروژه (سپهر) فارسی‌زبان است؛ توضیحات فارسی ساده.

## 🔑 اقتصاد توکن — بالاترین اولویت رفتاری

1. **نقشه قبل از جستجو:** اگر `graphify-out/GRAPH_REPORT.md` وجود دارد، قبل از هر Glob/Grep/خواندن فایل، اول آن را بخوان و فقط سراغ فایل‌هایی برو که نقشه نشان می‌دهد. هرگز کل پروژه را اسکن نکن.
2. **خواندن هدفمند اسناد:** docs/DESIGN.md و docs/ROADMAP.md را فقط «بخش مرتبط با کار فعلی» بخوان، نه کل فایل را در هر سشن. PROGRESS.md کوچک است و کامل خوانده می‌شود.
3. **ویرایش جراحی:** فایل‌ها را با edit‌های کوچک تغییر بده؛ هرگز فایل کامل را بازنویسی/بازچاپ نکن مگر فایل جدید باشد.
4. **بدون تکرار:** کدی که کاربر می‌تواند در فایل ببیند را در چت تکرار نکن — فقط بگو چه تغییری کجا اعمال شد.
5. **گزارش‌های کوتاه:** گزارش وضعیت و توضیحات حداکثر چند جمله. پرحرفی = هدررفت توکن.
6. فایل‌های داخل `graphify-out/` را دستی ویرایش نکن؛ به‌روزرسانی گراف با hook خودکار انجام می‌شود.

## پروتکل شروع سشن

**گام صفر — تشخیص حالت:** فایل PROGRESS.md را بخوان.
- اگر بخش «مرحله ۰» تیک نخورده ← **حالت BOOTSTRAP** (پایین را اجرا کن)
- در غیر این صورت ← **حالت کار عادی**

### حالت BOOTSTRAP (فقط اولین اجرا — خودکار، بدون سؤال اضافه)
به ترتیب انجام بده و نتیجه هر قدم را یک‌خطی گزارش کن:
1. `git init` اگر ریپو نیست + ساخت `.gitignore` استاندارد Next.js
2. نصب اسکیل طراحی: `npx ui-ux-pro-max-cli init --ai claude` (اگر پوشه `.claude/skills/ui-ux-pro-max` وجود ندارد)
3. نصب Graphify: `pip install graphifyy` سپس `graphify install` و `graphify claude install`
4. ساخت `.graphifyignore` (محتوا: `.git/`، `node_modules/`، `.next/`، `out/`، `dist/`، `coverage/`، `.env*`، `*.log`، `public/fonts/`)
5. کامیت اولیه اسناد
6. سپس برو سراغ حالت کار عادی
- نکته: ساخت گراف (`graphify .`) و `graphify hook install` را **بعد از مرحله ۱** (وقتی کدی وجود دارد) انجام بده و در PROGRESS ثبت کن.
- اگر قدمی شکست خورد (مثلاً pip نبود)، مشکل و راه‌حل را یک‌خطی بگو و ادامه بده — بوت‌استرپ نباید متوقف شود.

### حالت کار عادی
1. PROGRESS.md ← وضعیت و کار بعدی
2. بخش مرتبط docs/ROADMAP.md ← جزئیات مرحله فعال
3. گزارش با این قالب و سپس توقف برای تأیید:
   ```
   📍 مرحله [X] — [نام] | ✅ آخرین: [...] | ▶️ بعدی: [...]
   برنامه: [۲-۳ جمله فارسی ساده]. شروع کنم؟
   ```

## پروتکل پایان هر کار

1. `npm run build` (سلامت static export)
2. به‌روزرسانی PROGRESS.md (تیک + تاریخ + یک خط)
3. کامیت conventional (`feat:`/`fix:`/`style:`) — hook گراف را خودش تازه می‌کند
4. گزارش یک‌خطی به کاربر

## استک فنی — جدیدترین نسخه‌ها، غیرقابل مذاکره

- **Next.js 16** (App Router) + **React 19** + **TypeScript** + **Tailwind CSS v4**
- Tailwind v4 = پیکربندی CSS-first با `@theme` در globals.css — **فایل tailwind.config وجود ندارد**؛ همه توکن‌ها CSS variable هستند
- `output: 'export'` (static) — دیپلوی: GitHub Pages + GitHub Actions
- انیمیشن: Framer Motion (پکیج `motion`)
- **ممنوع:** API routes سروری، ISR، middleware، هر چیز ناسازگار با export

## 🇮🇷 سازگاری کامل با شبکه ایران — قانون سخت

هیچ منبع runtime نباید از سرویس‌های تحریمی/فیلتر/کند در ایران لود شود:
- **فونت/CSS/JS/تصویر خارجی ممنوع** — همه‌چیز self-hosted داخل ریپو (فونت در `/public/fonts` با `next/font/local`)
- **ممنوع در کد:** Google Fonts، Google Analytics/Tag، reCAPTCHA، Firebase، gravatar، امبد YouTube (به‌جایش آپارات)، هر SDK آمریکایی تحریمی
- آنالیتیکس: self-hosted (مثل Umami روی هاست ایرانی) یا هیچ — تصمیمش در PROGRESS ثبت شود
- فرم/ایمیل/پرداخت: فقط سرویس‌های در دسترس ایران (درگاه: زرین‌پال و مشابه — فاز بعد)
- تست پذیرش هر صفحه: در DevTools همه requestها باید same-origin باشند
- نصب پکیج‌ها اگر خطای 403 تحریم داد: استفاده از میرور یا پروکسی npm را به کاربر پیشنهاد بده

## قوانین طلایی طراحی (کامل: docs/DESIGN.md)

1. فقط مونوکروم (`ink` تا `paper`) + رنگ سمانتیک فقط برای وضعیت‌ها
2. هیچ مقدار خام — فقط توکن‌های `@theme`. توکن نبود؟ اول به DESIGN.md و `@theme` اضافه کن
3. RTL-first: فقط `ms/me` و `start/end`
4. فضای خالی سخاوتمندانه؛ انیمیشن ظریف + `prefers-reduced-motion`
5. قبل از کار UI: اسکیل ui-ux-pro-max را فعال کن
